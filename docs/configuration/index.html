Configuration

Configuration documentation

This document covers every public tuning surface in saddle-rendering-trail.

Normalized length always runs from the oldest visible point (0.0) to the newest point near the source (1.0). Normalized age runs from newly emitted (0.0) to expired (1.0).

TrailScalarCurve

TrailScalarCurve is a sorted list of TrailScalarKey { position, value }.

  • Positions are expected in 0.0..=1.0.
  • The runtime clamps evaluation outside the authored range.
  • Empty curves are repaired to a constant 1.0.

Practical guidance:

  • Use a rising tail-to-head width curve for contrails.
  • Use a bell-ish curve for melee swipes by adding a few interior keys.
  • Use alpha_over_age instead of shrinking lifetime_secs when you want a soft dissolve without shortening the trail too aggressively.

TrailGradient

TrailGradient is a sorted list of TrailColorKey { position, color }.

  • Positions are expected in 0.0..=1.0.
  • Colors interpolate in linear space.
  • Empty gradients are repaired to solid white.

Practical guidance:

  • Tail-dark, head-bright ramps read well for speed lines.
  • Tail-transparent, head-bright ramps are often better handled by keeping the color bright and shaping opacity with alpha_over_length.

TrailCustomMaterial

Attach TrailCustomMaterial(handle) to a trail source entity to override the auto-generated StandardMaterial. When present, the crate skips creating and syncing its own material and instead assigns your handle to the render entity.

  • Adding the component at runtime swaps the material immediately
  • Removing the component reverts to the auto-generated material from TrailMaterial
  • Changing the handle inside the component hot-swaps the material on the render entity

TrailMaterial3d<M>

Attach TrailMaterial3d(handle) to a trail source entity to use any Material type (e.g., ExtendedMaterial<StandardMaterial, MyExt>). You must also add TrailMaterialPlugin::<M>::new(Update) to your app so the handle is synced to the render entity.

When present, the StandardMaterial on the render entity is replaced with MeshMaterial3d::<M>. Vertex colors and UVs are still generated by the trail system — your material should sample ATTRIBUTE_COLOR and ATTRIBUTE_UV_0.

TrailStyleOverride

Attach TrailStyleOverride(style) to a trail source entity to override Trail::style. When present, the embedded style is ignored. Removing the component reverts to Trail::style. This is useful for sharing a single style definition across many trails.

TrailHistory

Present on every trail source entity after the first tick. Provides read and write access to the trail's sample point buffer.

MethodReturnsNotes
points()&[TrailSamplePoint]Current sample points, oldest first
points_mut()&mut Vec<TrailSamplePoint>Mutable access; marks the history as dirty
total_length()f32Polyline length of the current trail
len()usizeNumber of active points
is_empty()boolWhether the trail has no points
normalized_lengths()Vec<f32>Parallel to points(), 0.0 at oldest, 1.0 at newest

User modifier systems should run in TrailSystems::Modify (between Sample and BuildMesh).

TrailSystems::Modify

An empty system set between Sample and BuildMesh for user-defined modifier systems. Systems in this set can read/write TrailHistory to implement:

  • Wind displacement
  • Surface snapping
  • Noise perturbation
  • Hit detection from trail geometry

TrailLod

Attach TrailLod to a trail source entity for distance-based level of detail.

FieldTypeDefaultExpected rangeEffect
start_distancef3220.0>= 0.0Distance at which point-count reduction begins
end_distancef3260.0> start_distance recommendedDistance at which the trail reaches minimum detail
min_points_fractionf320.250.0..=1.0 recommendedFraction of Trail::max_points retained at or beyond end_distance

The effective max-points is linearly interpolated between Trail::max_points (at or below start_distance) and Trail::max_points × min_points_fraction (at or beyond end_distance). The distance is measured against the trail's resolved TrailViewSource, not always the shared active camera.

TrailSamplePoint

TrailSamplePoint is the public type for individual trail sample points. It exposes position, rotation, age, and optional per-point overrides.

FieldTypeDefaultNotes

Per-point overrides (custom_width, custom_color) are applied in the mesh builder. Set them in a modifier system running in TrailSystems::Modify via TrailHistory::points_mut().

TrailDebugSettings

Debug drawing is only active when the app includes Bevy gizmos and enabled = true.

TrailDebugSettings is both a Resource (global default) and a Component (per-entity override). When attached to a trail source entity, the per-entity settings take precedence over the global resource.
